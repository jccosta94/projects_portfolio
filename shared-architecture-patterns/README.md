# Shared Architecture Patterns

🚧 **Pattern library in progress.** Documented patterns landing soon.

Cross-project architecture patterns I keep coming back to — extracted from production systems and re-usable across AI-native deployments. Each pattern documented as a standalone reference with concrete examples drawn from the projects in this portfolio.

**Patterns being documented:**
- **AI routing patterns** — when to route to which model, cost-aware fan-out, fallback chains, tier-based routing (premium API vs subscription)
- **Multi-agent design** — hierarchical org-chart-shaped teams, dispatch authorization graphs, terminal vs recursive nodes, file-ownership boundaries
- **Orchestration philosophy** — when to use a team-of-agents vs a single dispatcher, when each pattern breaks, sequencing decisions
- **Cost-vs-quality tradeoffs** — model tier as a make-vs-buy decision, when premium is load-bearing vs decorative
- **Enterprise AI governance** — consent gates, audit ledgers, redaction proxies, advisory-only patterns, AI Policy Manager centerpieces
- **Evaluation and observability** — eval harnesses, run-logs, reproducible-prompt archives, per-capability dashboards
